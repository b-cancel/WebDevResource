Creating Feature Filled Forms

Most Frameworks have some form of form component, but most of these times they just provide basic functionality. But, just because the functionality isn't added by the framework makers doesn't mean it can't make a significant impact on your user experience. Filling out forms is a boring yet nessesary process so the least you can do to try to get your users through it is to make it as easy to use as possible. Helping the user skip steps when possible. That Exactly why I generated these guidelines, to help anyone in any frame work keep in mind the features that most users expect... or at the very least, most users realize when these are missing subconciously...

Feature Set

  • Automatic Scrolling Focus For Each Field
    • when you tap on a field
      • where the keyboard is closed
        • and the field is focused [NOT POSSIBLE]
        • and the field is unfocused
      • where the keyboard is opened
        • and the field is focused
        • and the field is unfocused
    • when a new error appears
    • when you start typing in a field
  • Per Field Input Validation
    • each field has its own error generator
    • each error generate can generate multiple errors
  • Submit Form After All Fields Validated
  • Tapping anything but a field unfocuses all fields
  • Refocusing on a field re-opens the keyboard if needed
  • Senstivie data hidden with option to show/hide
    • option to automatically hide when field unfocused
    • option to automatically show when field focused
  • label to show at all times the purose of the field
  • placeholder to show an example of what should be placed in the field or instructions
  • text on bottom right, can be used as hint
  • text on bottom left, can be used as a counter to show how close you are to the requirements
